# Часть 1. Проектирование и анализ — WeatherMemes

Цель этой части — показать понимание задачи, структуру приложения и продуманные решения до этапа реализации.

## 1.1. Пользовательские сценарии

1) **Просмотр погоды и мема**

Как пользователь, я хочу ввести город (или использовать геолокацию), чтобы получить текущую погоду и мем, соответствующий погодным условиям.

2) **Добавление собственного контента**

Как пользователь, я хочу добавить свой мем по URL или загрузить изображение с устройства, чтобы пополнить базу мемов для разных типов погоды.

3) **Голосование и шаринг**

Как авторизованный пользователь, я хочу ставить лайк/дизлайк мемам и делиться карточкой (мем + погода), чтобы отправлять друзьям актуальный и смешной контент.

## 1.2. Функциональные требования

### Обязательные

- Получение текущей погоды по городу и по координатам (геолокация).
- Определение категории погоды на бэкенде: `hot`, `cold`, `rain`, `snow`, `wind`, `cloudy`, `clear`.
- Показ карточки погоды: город/страна, иконка состояния, температура, скорость ветра, влажность, давление.
- Получение и отображение мема по категории погоды.
- Регистрация, вход, выход, проверка активной сессии.
- Ограничение голосования: один пользователь может проголосовать за мем только один раз.
- Добавление мема только авторизованным пользователем.
- Добавление мема двумя способами: URL изображения или выбор файла с устройства.
- Формирование и копирование share-card в буфер обмена.
- Переключение языка интерфейса RU/EN с сохранением выбора.

### Опциональные

- Взвешенный показ мемов (популярные показываются чаще).
- Fallback при копировании: если изображение нельзя скопировать, копируется текст.
- Fallback при ошибке загрузки мема: показать ссылку на изображение.

## 1.3. Проектирование API

Бэкенд расположен под префиксом `/api`.

### Weather

#### `GET /api/weather`

**Параметры запроса** (один из вариантов):

- `city` (string) — поиск по городу  
или
- `lat` (number), `lon` (number) — поиск по координатам  

Дополнительно:
- `lang` (`ru` | `en`) — язык погодного описания (по умолчанию `ru`).

**Ответ 200 (пример):**

```json
{
  "city": "Moscow",
  "country": "RU",
  "coord": { "lat": 55.75, "lon": 37.62 },
  "temp": 3,
  "feels_like": 1,
  "temp_min": 2,
  "temp_max": 4,
  "humidity": 80,
  "pressure": 1007,
  "visibility": 10000,
  "wind_speed": 5.2,
  "wind_deg": 210,
  "wind_gust": null,
  "clouds": 75,
  "description": "облачно с прояснениями",
  "icon": "04d",
  "sunrise": 1710901200,
  "sunset": 1710945300,
  "dt": 1710920000,
  "timezone": 10800,
  "category": "cloudy"
}
```

**Ошибки:**

- `400` — некорректные параметры (`city` или `lat/lon` отсутствуют/невалидны).
- `404` — город не найден.
- `500` — не настроен `OWM_API_KEY`.
- `502` — ошибка внешнего погодного сервиса.

### Memes

#### `GET /api/memes?category={category}`

Получить случайный мем выбранной категории (с учетом веса по рейтингу).

**Параметры:**
- `category` (опционально) — одно из: `hot`, `cold`, `rain`, `snow`, `wind`, `cloudy`, `clear`.

**Ответ 200:**

```json
{
  "id": 12,
  "url": "https://api.memegen.link/images/drake/clear_sky/go_outside.png",
  "category": "clear",
  "likes": 5,
  "dislikes": 1
}
```

**Ошибки:**
- `400` — невалидная категория.
- `404` — мемы не найдены.

#### `POST /api/memes`

Добавить новый мем.  
**Требует авторизацию**: `Authorization: Bearer <token>`.

**Тело запроса:**

```json
{
  "url": "https://example.com/meme.jpg",
  "category": "rain"
}
```

`url` может быть как обычным URL, так и `data:` URL (при загрузке файла с устройства).

**Ответ 201:** созданный объект мема.

**Ошибки:**
- `401` — отсутствует/невалидный токен.
- `400` — не заполнены `url`/`category` или невалидная категория.
- `500` — ошибка сохранения.

#### `POST /api/memes/:id/vote`

Поставить лайк/дизлайк.  
**Требует авторизацию**: `Authorization: Bearer <token>`.

**Тело запроса:**

```json
{ "vote": "like" }
```

Допустимые значения: `like` | `dislike`.

**Ответ 200:**

```json
{ "likes": 10, "dislikes": 2 }
```

**Ошибки:**
- `401` — отсутствует/невалидный токен.
- `404` — мем не найден.
- `409` — пользователь уже голосовал за этот мем.
- `400` — невалидное значение `vote`.

### Auth

#### `POST /api/auth/register`

Регистрация пользователя.

**Тело запроса:**

```json
{
  "username": "kirill",
  "password": "secret123"
}
```

Ограничения: `username >= 3`, `password >= 6`.

**Ответ 201:**

```json
{
  "token": "session_token",
  "user": { "id": 1, "username": "kirill" }
}
```

#### `POST /api/auth/login`

Вход пользователя.

**Тело запроса:**

```json
{
  "username": "kirill",
  "password": "secret123"
}
```

**Ответ 200:** тот же формат `{ token, user }`.

#### `GET /api/auth/me`

Проверка текущей сессии.  
**Требует авторизацию**: `Authorization: Bearer <token>`.

**Ответ 200:**

```json
{ "user": { "id": 1, "username": "kirill" } }
```

#### `POST /api/auth/logout`

Выход (удаление сессии).  
**Требует авторизацию**: `Authorization: Bearer <token>`.

**Ответ 204:** без тела.

## 1.4. Модель данных

Используется SQLite. Основные сущности:

### Таблица `memes`

| Поле | Тип | Ограничения | Описание |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | Идентификатор мема |
| url | TEXT | NOT NULL | URL или data URL изображения |
| category | TEXT | NOT NULL, CHECK IN (...) | Категория погоды |
| likes | INTEGER | DEFAULT 0 | Кол-во лайков |
| dislikes | INTEGER | DEFAULT 0 | Кол-во дизлайков |
| created_at | TEXT | DEFAULT datetime('now') | Время добавления |

### Таблица `users`

| Поле | Тип | Ограничения | Описание |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | Идентификатор пользователя |
| username | TEXT | UNIQUE, NOT NULL | Логин |
| password_hash | TEXT | NOT NULL | Хеш пароля (SHA-256) |
| created_at | TEXT | DEFAULT datetime('now') | Время регистрации |

### Таблица `sessions`

| Поле | Тип | Ограничения | Описание |
|---|---|---|---|
| token | TEXT | PK | Токен сессии |
| user_id | INTEGER | NOT NULL, FK users(id) | Владелец токена |
| created_at | TEXT | DEFAULT datetime('now') | Время создания |

### Таблица `meme_votes`

| Поле | Тип | Ограничения | Описание |
|---|---|---|---|
| meme_id | INTEGER | NOT NULL, FK memes(id) | Мем |
| user_id | INTEGER | NOT NULL, FK users(id) | Пользователь |
| vote | TEXT | NOT NULL, CHECK IN ('like','dislike') | Тип голоса |
| created_at | TEXT | DEFAULT datetime('now') | Время голосования |

Ключевая защита от накрутки:

- `PRIMARY KEY (meme_id, user_id)` в `meme_votes` — не дает голосовать повторно за тот же мем одним пользователем.

## 1.5. Ключевые технические решения

### Стек и обоснование

- **Frontend: React + TypeScript + Vite**
  - быстрое прототипирование и разработка UI;
  - строгая типизация и устойчивость к ошибкам;
  - удобная модульная структура компонентов.

- **Backend: Node.js + Express + TypeScript**
  - простой REST API;
  - единый язык с frontend (TS) для снижения когнитивной нагрузки;
  - предсказуемая структура роутов и middleware.

- **База: SQLite (`better-sqlite3`)**
  - не требует отдельного сервера;
  - подходит для учебного/демо-проекта;
  - быстрый старт и простое резервирование.

### Внешние API и сервисы

- **OpenWeatherMap API**
  - получение текущей погоды по городу и координатам;
  - иконка состояния и текстовое описание погоды;
  - поддержка параметра языка (`lang=ru|en`).

### Архитектурные решения

- Категоризация погоды выполняется на бэкенде, чтобы логика была едина для всех клиентов.
- Авторизация на основе сессионного Bearer token.
- Share-card формируется на frontend через `canvas` и копируется в буфер обмена как изображение.
- Локализация интерфейса RU/EN хранится в `localStorage`, чтобы сохранять выбор пользователя.
